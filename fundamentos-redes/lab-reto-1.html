Laboratorio Práctico: Reto 1

Anatomía de una Red Doméstica y Diagnóstico

Módulo: Fundamentos de Redes y Arquitecturas

BRIEFING DE LA MISIÓN

Tu objetivo es construir y validar una red doméstica básica utilizando Packet Tracer. A través de este proceso, aplicarás conceptos fundamentales de topología, direccionamiento IP y herramientas de diagnóstico. Sigue las fases en orden. El éxito en cada fase desbloquea la siguiente.

Fase 1: Reconocimiento y Construcción Inicial

Identifica los componentes esenciales y establece la conexión básica a Internet.

> Instrucciones para Packet Tracer:

  1. Añade un 'PC' (End Devices).
  2. Añade un 'Router-PT' (Routers).
  3. Añade una 'Cloud-PT' (WAN).
  4. Conecta el PC al Router (FastEthernet0/0) y el Router (FastEthernet1/0) a la Cloud usando el cable de cobre directo (Copper Straight-Through).

Pregunta 1: Para que el PC pueda comunicarse con internet, el router debe realizar una función clave que "traduce" la dirección IP privada del PC a una dirección IP pública. ¿Cómo se llama esta función?

Fase 2: Expansión de la Red Local (LAN)

Añade más dispositivos y configúralos para que coexistan en la misma red.

> Instrucciones para Packet Tracer:

  1. Añade un 'Switch-PT' (Switches) y conéctalo al Router (FastEthernet0/0).
  2. Desconecta el PC del Router y conéctalo al Switch.
  3. Añade un 'Server-PT' (End Devices) y conéctalo al Switch.
  4. Configura IPs estáticas: PC (192.168.1.10) y Servidor (192.168.1.11). Máscara de subred: 255.255.255.0. Gateway: 192.168.1.1 (la IP del router en la LAN).

Pregunta 2: El switch opera principalmente en una capa del modelo OSI para dirigir el tráfico basándose en direcciones físicas (MAC). ¿Cuál es esa capa?

Fase 3: Verificación de Conectividad

Utiliza herramientas de diagnóstico para confirmar que tu red funciona como se espera.

> Instrucciones para Packet Tracer:

  1. Abre la terminal del PC (Desktop > Command Prompt).
  2. Ejecuta el comando: ping 192.168.1.11. Deberías recibir 4 respuestas exitosas.

Pregunta 3: El comando ping utiliza un protocolo específico para enviar sus "echo requests" y recibir "echo replies", operando en la capa de Red. ¿Qué protocolo es?

LABORATORIO COMPLETADO

Has construido, configurado y verificado con éxito una red básica. Has demostrado comprender la función de los componentes clave, el direccionamiento IP y las herramientas de diagnóstico fundamentales. Tu red está operativa.

Este es el fundamento. En los próximos retos, introduciremos más complejidad, más dispositivos y, por supuesto, más amenazas que defender.

Este laboratorio es una simulación de entrenamiento. Las configuraciones son simplificadas con fines educativos.
